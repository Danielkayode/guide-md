Overview

GUIDE.md is a YAML-frontmatter markdown file that defines how AI agents should interact with your codebase. It provides context, constraints, and guardrails to ensure consistent AI behavior across different tools and models.

File Structure

---
# YAML frontmatter
key: value
---

# Markdown body
## Project Overview
...

## Domain Vocabulary
...

Required Fields

Must follow semver format: "1.0.0"

guide_version: "1.0.0"
project Required string
Min 2 characters, max 100 characters
The name of your project.
project: "my-awesome-app"
language Required string | string[]
One or more supported languages
Primary programming language(s) for the project.
Supported: typescript, javascript, python, rust, go, java, kotlin, swift, cpp, c, csharp, ruby, php, scala, haskell, elixir, zig

strict_typing Required boolean
Instructs AI to always use explicit types; never 'any' or untyped params.
strict_typing: true
error_protocol Required enum
One of: "verbose", "silent", "structured"
How AI should handle and surface errors in generated code.
error_protocol: verbose

Nested Schemas

code_style

Code style preferences for AI-generated code.
  • max_line_length (number, default: 100) — Max characters per line
  • indentation (string, default: "2 spaces") — Use "tabs" or "N spaces"
  • naming_convention (enum, default: "camelCase") — camelCase, snake_case, PascalCase, kebab-case, SCREAMING_SNAKE
  • max_function_lines (number, optional) — Soft limit for function length
  • prefer_immutability (boolean, default: false)
  • prefer_early_returns (boolean, default: true)

guardrails

Safety controls for AI behavior.
  • no_hallucination (boolean, default: true) — Don't invent APIs or types
  • cite_sources (boolean, default: false) — Cite docs for unfamiliar APIs
  • scope_creep_prevention (boolean, default: true) — Only modify referenced files
  • dry_run_on_destructive (boolean, default: false) — Preview destructive ops
  • max_response_scope (enum, default: "function") — file, function, class, module

testing

Testing requirements.
  • required (boolean, default: false) — Tests required for changes
  • framework (string, optional) — Testing framework name
  • coverage_threshold (number, optional) — Minimum coverage % (0-100)
  • test_alongside_code (boolean, default: false) — Generate tests with code

context

Project structure context.
  • entry_points (string[]) — Key files for AI navigation
  • off_limits (string[]) — Glob patterns AI should never modify
  • architecture_pattern (enum, optional) — mvc, clean, hexagonal, layered, microservices, monolith, serverless, event-driven
  • state_management (string, optional) — State management approach

Inheritance

The extends field allows GUIDE.md files to inherit from other files:

  • Local files: extends: "./base-guide.md"
  • Registry modules: extends: "typescript-strict"
  • Remote URLs: extends: "https://example.com/guide.md"

Child values override parent values. Arrays are replaced, not merged. Nested objects are deeply merged.

Body Sections

The markdown body should include these sections:

  • Project Overview — Describe purpose, domain context, business rules
  • Domain Vocabulary — Define key terms for consistent naming
  • Non-Obvious Decisions — Explain unusual architectural choices
  • What NOT to do — Anti-patterns specific to this codebase
